## `zoa-factors` — actively maintained

Daily reference/metadata bucket. Updated every morning by a remote pipeline (server-side,
not in local repos). Original logic lives in `/Users/brian/notebooks/`. The Rust feature
engineering code in `research/paratrade/libs/feature-eng/src/factors.rs` reads from this
bucket at runtime.

### Files

**`symbols.parquet`** (date, symbol, type)
- All symbols trading on each day since 2018-05-01. Types: `Equity`, `Etf`, `Adr` only —
  no ETN. Source: Databento ARCX.PILLAR definition schema, processed in
  `/Users/brian/notebooks/metadata_gen_new.ipynb`.
- Use for: survivorship-bias-free universe construction.

**`tiingo_eod.parquet`** (symbol, date, open, high, low, close, adjClose, adjOpen, adjHigh,
adjLow, volume, adjVolume, splitFactor, divCash)
- Complete daily OHLCV from Tiingo API, all history, all symbols. Full file rebuilt each
  morning (not append-only). Split- and dividend-adjusted fields provided natively by Tiingo.
  `adj_prev_close = close.shift(1) / splitFactor - divCash` for consistent return series.
- Generation code: logic in `/Users/brian/notebooks/metadata_tiingo.ipynb`, runs on remote server.
- Use for: open-to-close return targets (`adjClose / adjOpen - 1`).

**`vnv.parquet`** (date, symbol, open, close, avg_volume_dollars, avg_volume_dollars_pctl,
daily_log_volatility)
- Per-symbol daily stats derived from `tiingo_eod.parquet`. Generation code:
  `notebooks/metadata_tiingo.ipynb`.
- `avg_volume_dollars`: 100-trading-day rolling mean of `volume * (open+close)/2`, shifted
  1 day (excludes current day). min_samples=10.
- `avg_volume_dollars_pctl`: cross-sectional percentile rank on each date, computed only
  among Equities with `avg_volume_dollars > $10M`. Symbols below $10M get null.
- `daily_log_volatility`: 100-trading-day rolling std of `log(close / adj_prev_close)`,
  shifted 1 day. min_samples=10.

**`high_adv_symbols.parquet`** (same as vnv + `type` column) — **deprecated, still updating**
- Subset of `vnv.parquet` filtered to high-ADV symbols, joined with `symbols.parquet` for
  the `type` field. Exact percentile cutoff unknown — generated by remote pipeline, write
  statement not present in local code. Used by
  `/Users/brian/research/paratrade/libs/feature-eng/src/bin/run_transformer_features.rs`.
- Still updated daily but considered deprecated. Prefer filtering `vnv.parquet` directly
  by `avg_volume_dollars_pctl` for new code.

**`spy_betas.parquet`** (date, symbol, spy/qqq/xbi_covar, spy/qqq/xbi_var, rolling versions,
spy_beta, qqq_beta, xbi_beta)
- Per-symbol daily and rolling betas to SPY, QQQ, and XBI. Fully self-computed from
  `consol_1m` data — no external source. Generation code: `/Users/brian/notebooks/betas.ipynb`.
- Method: sample 15-min log-return windows every 5 minutes between 10:00am–3:30pm ET using
  1-min mid prices. Outliers clipped at 4× daily stdev before computing.
- `spy_covar/spy_var`: single-day sum of (stock × SPY log-ret products) and (SPY log-ret²).
- `spy_beta` (rolling): 100-trading-day rolling sum of covar / rolling sum of var, shifted 1
  day forward. min_samples=50. This is a rolling OLS regression through the origin.
- Same calculation for QQQ and XBI.

**`consol_1m/{date}.parquet`** (date, symbol, timestamp, nbb_px, nbo_px, nbb_sz, nbo_sz)
- 1-minute consolidated NBBO (best bid/ask price and size across all exchanges). Regular
  hours only (9:30–16:00 ET). 2018-05-01 → today, updated daily.
- Source: NasdaqBasic consolidated feed.
- Use for: intraday feature engineering, beta computation, spread/momentum features.

**`per-day/{date}/vnv.parquet`**
- Same data as root `vnv.parquet` but partitioned by day for fast single-day lookups.

**`backups/{date}/`**
- Daily snapshots of all root-level files (tiingo_eod, vnv, spy_betas, symbols,
  high_adv_symbols) for point-in-time reconstruction.

## `paradome-trf` — actively maintained

One parquet file per day: `{YYYY-MM-DD}.parquet`. 2018-05-01 → today, updated daily ~9am ET.
Recent files ~800MB–900MB/day, ~58M rows/day.

**What TRF is:** FINRA's Trade Reporting Facility — all trades that happen off-exchange (dark
pools, internalized retail flow, OTC institutional blocks) must be reported here within seconds.
Roughly 35–40% of US equity volume is off-exchange. Critical signal source for order flow.

**Schema:**

| Column | Meaning |
|---|---|
| `symbol` | Ticker |
| `timestamp` | Millisecond precision, ET timezone, full extended hours |
| `price` / `quantity` | Trade price and shares |
| `sales_conditions` | FINRA sale condition codes (see below) |
| `market_center` | Always `D` — FINRA TRF market center code |
| `sub_market_center` | `B` = FINRA TRF/Nasdaq, `Q` = FINRA TRF/NYSE |
| `canceled_indicator` | Whether trade was subsequently busted/canceled |
| `upstream_seq` / `msn` / `omsn` | Sequence numbers for deduplication |
| `trade_date` | Unix timestamp of trading date |
| `date` | Trading date as date32 |

**Key sales_conditions values:**
- `@ T`  — regular extended-hours trade, last-sale eligible
- `@ TI` — extended-hours odd-lot (< 100 shares), last-sale eligible
- `  T`  — extended-hours trade, not last-sale eligible (late report)
- `  TI` — extended-hours odd-lot, not last-sale eligible
- `@` prefix = qualifies as last reported sale. `T` = extended hours. `I` = odd-lot.

**How it's used:** Read by the Rust feature engineering pipeline. Replays trades
chronologically via a `ManualClock` callback to compute TRF flow signals:
`trf_intraspread_volume_signed_norm`, `trf_cbbo_volume_signed_norm`, etc.
Reader code: `/Users/brian/research/paratrade/libs/trf/src/lib.rs`.

**Source:** Databento XNAS.BASIC feed, FINRA TRF publisher IDs (`FINN`, `FINC`), processed
and stored as parquet on the remote server.

## `zoa-lseg-prs` — actively maintained (pending deprecation)

Daily LSEG (Refinitiv) Newswire press release stories as JSONL. Key format:
`STORY.ZOA.{YYYY-MM-DD}.jsonl`. 2026-01-13 → today. Updated daily ~7pm ET.
~20–30MB/day, ~3,200 stories/day.

**Deprecation notice:** This bucket will eventually be merged into `zoa-clean-mrns` and
deprecated. Use `zoa-clean-mrns` for all history pre-2026; use this bucket for recents
until the migration is complete.

**Format:** Multi-line JSON objects (not newline-delimited). Each record is a full JSON
object starting with `{"guid":...}`. Parse with `json.JSONDecoder().raw_decode()` in a loop.

**Top-level fields:**
- `guid` — unique story ID
- `zoa_received_at` — when we received it (UTC)
- `v_maj` / `v_min` — schema version
- `data` — the story payload

**`data` fields:**
- `headline` — story headline
- `body` — full text body
- `firstCreated` / `versionCreated` — story timestamps (UTC ISO8601)
- `provider` — wire service: `NS:PRN` (PR Newswire), `NS:BSW` (Business Wire), `NS:GNW` (GlobeNewswire)
- `subjects` — LSEG topic/instrument codes. `R:TICKER.X` entries are RIC codes linking to specific stocks
- `audiences` — distribution channels
- `language` — story language (mostly `en`, some `de`, `pt`, `zh`, etc.)
- `urgency` — numeric priority (3 = normal)
- `pubStatus` — `stat:usable` for live stories
- `mimeType` — `text/plain`

**RIC codes** (in `subjects` array, prefix `R:`) link stories to specific equities.
Format: `R:TICKER.X` where `.N` = NYSE, `.O` = NASDAQ, `.TO` = TSX, `.HK` = HK, etc.
Not all stories have RICs — press releases often don't tag a stock.

**Use for:** NLP-based event features — earnings announcements, offering alerts, M&A,
law firm class action filings, FDA/regulatory news. Useful for daily alpha signals but
requires NLP processing to extract signal. No structured numeric data.

## `zoa-mbp1-daily` — actively maintained

Daily MBP-1 (market-by-price, top-of-book) quote data from Databento, one file per
exchange per day. Key format: `{exchange}/{YYYYMMDD}.mbp-1.dbn.zst`. Updated daily.
Files are ~11–18 GB each (uncompressed equivalent). Stored in Databento's native DBN
binary format, zstd-compressed.

**What MBP-1 is:** Market-by-price, level 1 — the best bid and ask on a single exchange
after every top-of-book change. Unlike NBBO (consolidated across exchanges), each file is
exchange-specific. Unlike MBO (market-by-order), it only stores the top-of-book price
level, not individual order IDs. More granular than 1-min bars; less granular than full
order book.

**Exchanges (13 total):**
`arcx.pillar`, `bats.pitch`, `baty.pitch`, `edga.pitch`, `edgx.pitch`, `eprl.dom`,
`memx.memoir`, `xase.pillar`, `xbos.itch`, `xchi.pillar`, `xnas.itch`, `xnys.pillar`,
`xpsx.itch`

**Date range:** 2018-05-01 → today.

**Use for:** Sub-minute bid/ask reconstruction per exchange. Building consolidated NBBO
from scratch. Spread features, mid-price series, liquidity analysis. Source data for
`consol_1m` generation (the consolidated 1-min NBBO in `zoa-factors`).

**Note:** Raw DBN files require `databento` Python library or `dbn` crate to decode. Not
directly readable as parquet. For most feature engineering use `consol_1m` (already
consolidated and in parquet) instead of the raw per-exchange MBP-1 files.

## `zoa-daily-per-symbol-mbo-{year}` (2018–2026) — actively maintained (2026 bucket)

Per-symbol MBO (market-by-order) tick data, one bucket per calendar year. Key format:
`{exchange}/{YYYYMMDD}/{symbol}.mbo.dbn.zst`. Stored in Databento's native DBN binary
format, zstd-compressed.

**What MBO is:** Full order book event stream — every add, modify, cancel, and fill for
every individual order. Not just trades — most events are order queue changes that never
execute. Fills (executed orders) are a subset. Lets you reconstruct the full limit order
book depth at any timestamp.

**Contrast with other data:**
- `paradome-trf` — completed off-exchange trades only (dark pools, internalized flow)
- `zoa-mbp1-daily` — per-exchange top-of-book snapshots after each change (no order IDs)
- `zoa-daily-per-symbol-mbo-*` — every on-exchange order lifecycle event, full depth

**Buckets:**
- `zoa-daily-per-symbol-mbo-2018` through `zoa-daily-per-symbol-mbo-2025` — static
  complete-year archives. Bucket creation dates all ~2025-06-23, suggesting a bulk
  historical load.
- `zoa-daily-per-symbol-mbo-2026` — **actively updated daily** (current year).

**Exchanges (same 13 as zoa-mbp1-daily):**
`arcx.pillar`, `bats.pitch`, `baty.pitch`, `edga.pitch`, `edgx.pitch`, `eprl.dom`,
`memx.memoir`, `xase.pillar`, `xbos.itch`, `xchi.pillar`, `xnas.itch`, `xnys.pillar`,
`xpsx.itch`

**Reader:** `paratrade_py.get_book_states()` — decodes DBN, reconstructs book state at
requested timestamps. Raw files require `databento` library or `dbn` crate to decode.

(which has 2018–today as full-exchange files).

## `zoa-daily-training-data-xnas-v2` (historical, stopped Jun 2025)

Pre-computed second-level intraday ML features for the existing intraday GBM pipeline.
Key format: `{YYYY-MM-DD}_{shard}.parquet`. 10 shards per day, ~275 MB each (~2.75 GB/day).
Date range: 2018-07-12 → 2025-06-12. **No longer updating.**

Each shard covers ~100 symbols for a full trading day. One row per (symbol, second-level
timestamp), regular hours only (9:30–16:00 ET). ~2.3M rows per shard/day.

This is the data source for `notebooks/gbm_train.py` (intraday LightGBM).

**Schema (53 columns):**

| Column | Type | Meaning |
|---|---|---|
| `symbol` | string | Ticker |
| `date` | date32 | Trading date |
| `timestamp` | timestamp[ns] | Second-level UTC timestamp |
| `cbb` / `cbo` | double | Consolidated best bid / offer price |
| `cbb_size` / `cbo_size` | uint64 | Best bid / offer size |
| `swmid` | double | Size-weighted mid price |
| `spy_swmid` | double | SPY size-weighted mid (same timestamp) |
| `time_since_open_secs` | double | Seconds since 9:30am ET |
| `liquidity_pctl` | double | Cross-sectional ADV percentile (from vnv) |
| `sqrt_adv` | double | Square root of avg daily volume dollars |
| `mark_spy_hedged_swmid_norm_300` | double | SPY-hedged price mark (5-min) |

Then for each lookback window **60, 150, 300, 900, 1800 seconds**:

| Column pattern | Meaning |
|---|---|
| `return_swmid_norm_{N}` | Raw mid-price return over N seconds, normalized |
| `return_spy_hedged_swmid_norm_{N}` | SPY-hedged return over N seconds |
| `volume_norm_{N}` | Total on-exchange volume over N seconds, normalized |
| `cbbo_volume_signed_norm_{N}` | Net signed CBBO volume (buy minus sell), normalized |
| `cbbo_sweep_volume_signed_norm_{N}` | Signed sweep (aggressive) volume, normalized |
| `trf_volume_norm_{N}` | TRF (off-exchange) volume, normalized |
| `trf_intraspread_volume_signed_norm_{N}` | TRF volume inside the spread, signed |
| `trf_cbbo_volume_signed_norm_{N}` | TRF volume at CBBO prices, signed |

**Normalization:** All `_norm` fields are normalized by `sqrt_adv` (square root of ADV
dollars), making them comparable across stocks of different sizes.

**Note:** No target column in the file — target (`return_spy_hedged_swmid_norm2_f900`,
i.e. 15-min forward SPY-hedged return) is computed at training time by forward-shifting
`return_spy_hedged_swmid_norm_900`.

**Related stopped buckets (superseded versions):**
- `zoa-daily-training-data` — stopped Apr 2025 (earlier schema version)
- `zoa-daily-training-data-xnas` — stopped Apr 2025
- `zoa-daily-training-xnas-basic-fixed` — stopped Sep 2025 (small extension of xnas-v2)

## `zoa-cmbp1-daily` — actively maintained

Daily consolidated CBBO (consolidated best bid/offer) from NasdaqBasic. Key format:
`xnas-basic-{YYYYMMDD}.cmbp-1.dbn.zst`. Updated daily. Files range ~6–19 GB/day
(compressed). Date range: 2024-07-01 → today.

**What CMBP-1 is:** Consolidated market-by-price, level 1 — the national best bid and
offer (NBBO) after every top-of-book change across all exchanges, as reported by the
NasdaqBasic consolidated feed. This is the raw tick-level source for `consol_1m`
(the pre-aggregated 1-min NBBO parquets in `zoa-factors`).

**Relationship to other data:**
- `zoa-mbp1-daily` — per-exchange top-of-book (not consolidated)
- `zoa-cmbp1-daily` — consolidated NBBO tick stream (this bucket)
- `zoa-factors/consol_1m/{date}.parquet` — 1-min aggregated NBBO derived from this data

**Use for:** Sub-minute consolidated quote reconstruction. The `consol_1m` parquets are
derived from this data and cover the full history back to 2018. Use `consol_1m` for
feature engineering unless you need sub-minute resolution or pre-2024-07 tick data.

**Note:** Raw DBN format — requires `databento` library or `dbn` crate to decode. Not
directly readable as parquet.

## `zoa-fda-calendar` — actively maintained

Daily FDA event calendar as JSON. Key format: `fda_calendar_{YYYY-MM-DD}.json`. Updated
daily at ~1am ET. Date range: 2026-01-22 → today. Files ~3.5–3.7 MB/day, ~900 records/day.

**Format:** Top-level JSON array. Each element is one drug/catalyst event.

**Key fields per record:**

| Field | Meaning |
|---|---|
| `company_ticker` | Stock ticker of the company |
| `company_name` | Company name |
| `drug_name` | Drug/treatment name |
| `indication` | Disease/condition being treated |
| `simplified_stage` | Trial stage (`phase1`–`phase5`, `phase5` = approved/commercial) |
| `label` | Event type: `PDUFA` (FDA decision date), `AdCom`, `Data Readout`, etc. |
| `catalyst_date` | Expected date of the FDA event |
| `catalyst_date_text` | Text description if date is approximate |
| `fda_status_label` | Current status: `Ongoing`, `Approved`, `Complete Rejection`, etc. |
| `likelihood_of_approval` | Analyst consensus probability of approval (0–100) |
| `market_cap` / `volume` / `average_daily_volume` | Market data as of file date |
| `company_optionable` | Whether options trade on this stock |

**Use for:** Binary event signals — PDUFA dates are hard catalysts that move biotech stocks
significantly. Can build: (1) pre-event long/short signals based on approval probability,
(2) post-event return analysis, (3) universe filter to exclude stocks with imminent binary
events. Join to `symbols.parquet` or `tiingo_eod.parquet` via `company_ticker`.

**Note:** File is a snapshot of the entire forward-looking calendar as of that date — not
just that day's events. All upcoming catalysts are included. Filter by `catalyst_date` to
find events happening on a specific date.

## `zoa-clean-mrns` — actively maintained (currently historical archive through Nov 2025; will absorb `zoa-lseg-prs`)

Historical archive of LSEG MRN (Machine Readable News) — the broad Refinitiv newswire
feed. Key format: `{YYYYMMDD}.jsonl.zst` (true newline-delimited JSON, zstd-compressed).
Date range: 2018-01-01 → 2025-11-30. All files bulk-loaded 2026-01-21. **Not actively
updating** — `zoa-lseg-prs` (2026-01-13 → today) is the live successor for press releases.

Files ~1–40 MB/day compressed (~10,000 stories/day on active trading days, fewer on
weekends/holidays).

**Roadmap:** `zoa-lseg-prs` (the current live press release feed) will eventually be
merged into this bucket and deprecated. Build any pipelines to read from `zoa-clean-mrns`
for history and `zoa-lseg-prs` for recents, with the expectation that `zoa-lseg-prs`
goes away once this bucket is live again.

**Distinction from `zoa-lseg-prs`:** That bucket is filtered to press release providers
only (PR Newswire, Business Wire, GlobeNewswire). This bucket is the broader MRN feed
including hundreds of providers — wire services, regional outlets, international sources,
law firm filings, etc.

**Format:** Standard JSONL — one JSON object per line (unlike `zoa-lseg-prs` which uses
multi-line objects). Parse with `json.loads(line)`.

**Top-level fields:**
- `timestamps` — list of `{name, source, timestamp}` objects recording when the story was
  received at each Refinitiv regional node (AMER, APAC, EMEA)
- `data` — the story payload

**`data` fields:**

| Field | Meaning |
|---|---|
| `altId` | Alternate story ID |
| `id` | Primary story ID |
| `headline` | Story headline |
| `body` | Full text body |
| `firstCreated` / `versionCreated` | Story timestamps (UTC ISO8601) |
| `provider` | Wire service code (e.g. `NS:PRN`, `NS:BSW`, `NS:ACSWIR`, `NS:BSE`) |
| `subjects` | LSEG topic/instrument codes. `R:TICKER.X` = RIC code linking to a stock |
| `language` | Story language (`en`, `es`, `zh`, `ko`, `pt`, ...) |
| `urgency` | Numeric priority |
| `pubStatus` | `stat:usable` for live stories |
| `messageType` | Numeric message type |
| `takeSequence` | Version sequence number |
| `audiences` | Distribution channels |

**RIC codes** (in `subjects`, prefix `R:`) link stories to specific equities — same
format as `zoa-lseg-prs`. ~54% of stories have at least one RIC code.

**Top providers (from sample):** `NS:BSE` (Bombay SE), `NS:INFARH`, `NS:CNI`, `NS:HIIS`,
`NS:EDLY`, `NS:MTD` (Korean), `NS:ELCOME`, `NS:PRN` (PR Newswire), `NS:BSW` (Business
Wire), `NS:ACSWIR` (Accesswire), and many others.

**Top languages (from sample):** English (~55%), Spanish (~21%), Chinese (~13%), Korean
(~7%), Portuguese (~3%).

**Use for:** Long-horizon NLP features — earnings event detection, M&A rumors, regulatory
filings, law firm class actions, macro news. Filter to `language=en` and `R:*.N`/`R:*.O`
RIC codes for US equity signals. Covers 2018–2025, so usable for full historical backtest
feature generation.

## `zoa-va-backup` — actively maintained

Visible Alpha (VA) data backup. Two prefixes:
- `va_snapshots/` — per-symbol point-in-time snapshots. Key format:
  `{SYMBOL}_{timestamp}_{date}.json.zst`. Bulk-loaded 2026-01-26.
- `va_revisions/` — revision history.

other financial metrics, aggregated from sell-side analyst models. More granular than
